Historia
de la Web

  • HTML parte de SGML que ya era un estándar
  • la versión inicial de html tenía unos pocos elementos, p, ul, ol, h1 a h6 y a con href, que fue la invención de Tim
  • en la convención de 1993 se buscó crear un estándar de HTML para mejorar la interoperabilidad, es decir que funcione bien en todos lados
  • la internacionalización fue para permitir otros tipos de charsets dentro de html
  • en 2004 se forma el WHATWG que busca acelerar la implementación del estandar

Semántica HTML5

Significado de una página HTML

Se introdujo con la intención de darle un contexto más representativo a los contenidos de un sitio o aplicación web.

No es una cuestión caprichosa, ya que una semántica correcta y descriptiva mejora: ACCESIBILIDAD, BÚSQUEDAS, INTERNACIONALIZACIÓN e INTEROPERABILIDAD.

organizado el contenido de forma semántica, utilizando atributos para mejorar la accesibilidad de formularios, imagen, video sobre todo para gente con capacidades diferentes.

Content Models

Flow: a, article, button, div, fieldset, figure, section, casi todos los elementos Sectioning: delimitan el scope de títulos y footers Heading: h1 al h6, definen títulos de las secciones phrasing: contenido embedded: imagen, video, audio, iframes interactive: contenido específicamente hecho para interactuar con el usuario, a, audio, video, button

Introducción a la Accesibilidad (A11Y)

Permitir a la mayor cantidad de personas utilizar los websites inclusive cuando las capacidades de esas personas están limitadas de alguna manera.

  • utilizar la semántica correcta para garantizar funcionalidades de los elementos correctos, ej, usar un button en vez de un div

Tips de accesibilidad

  • Navegable con teclado
  • Alternativa textual para imágenes, audio y video
  • Cuidar el contraste de colores
  • Utilizar HMTL semántico
  • Formularios accesibles
  • Utilizar tablas sólo para mostrar datos tabulares

Optimización del HTML
